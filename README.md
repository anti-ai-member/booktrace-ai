# 书脉

> 读得清脉络，记得往来处。

[English README](./README.en.md)

书脉是一个面向深度阅读的本地优先阅读器原型。它不只负责把书打开，更关注读者在长篇阅读里最容易遇到的问题：人物太多、关系记不住、时间线混乱、地点和事件前后断裂，以及看到某段文字时无法快速回到出处和上下文。

项目当前优先打磨桌面端体验：安静、干净、阅读优先，同时把 AI 能力做成阅读过程里的自然辅助，而不是复杂的外部仪表盘。

## 核心能力

- **本地书架**：管理本地图书、分类标签、阅读进度和最近阅读位置。
- **EPUB 阅读器**：支持本地 EPUB 解析、目录跳转、分页阅读、阅读进度恢复。
- **AI 阅读索引**：基于已读内容提取人物、地点、时间线、关键事件和关系线索。
- **证据回链**：每个 AI 结论都保留原文证据位置，方便跳回出处。
- **优先级展示**：重大人物、关键地点、历史意义时间节点优先展示；没有重大项时展示最近已读内容。
- **增量分析**：默认只分析已读内容；再次分析时会读取上次结果并继续补充。
- **书签与笔记**：本地保存每本书的书签、笔记和阅读位置。
- **选中文本辅助菜单**：选中一段文字后弹出轻量操作菜单，目前已实现 UI、动画和笔记入口。
- **关系可视化**：当模型识别到可靠关系时，提供关系图、组织结构和矩阵视图。
- **阅读主题**：提供素笺、荷花、香茗、兰花、花枝、竹林等雅淡主题。

## AI 设计原则

书脉的 AI 不是替读者读完整本书，而是跟随阅读进度建立“已读范围内”的记忆索引：

1. 只分析读者已经读到的内容，避免剧透。
2. 每次分析都继承上次结果，只补充、修正新增已读内容。
3. 人物、地点、时间线必须和主线密切相关。
4. 人物出生日期、出版信息、顺带提到的历史年份等不会进入主时间线。
5. 地点列表展示地点简介，不把原文摘录直接当作简介。
6. 所有结论都尽量保留原文证据，便于回查。

## 当前支持格式

当前可直接阅读：

- EPUB

已预留导入入口，后续计划接入解析器：

- PDF
- TXT
- HTML
- RTF
- DOC / DOCX
- MOBI
- AZW / AZW3
- FB2
- DjVu
- CBZ / CBR

- LangChain

## 本地运行

```bash
npm install
npm run dev
```

默认会同时启动：

- 前端：`http://localhost:5173`
- 本地 AI API：`http://127.0.0.1:8787`

构建生产版本：

```bash
npm run build
```

预览构建结果：

```bash
npm run preview
```

## AI 配置

复制环境变量示例：

```bash
cp .env.example .env
```

填写至少一个 API Key：

```env
DEEPSEEK_API_KEY=
OPENAI_API_KEY=
```

默认模型：

- Provider：DeepSeek
- Model：`deepseek-v4-flash`

也可以在阅读器的 AI 设置里切换到 OpenAI 模型。API Key 只从本地 `.env` 读取，不会进入浏览器端状态。

## 发展路线

- 接入 PDF、TXT、MOBI、AZW3 等更多格式的真实解析能力。
- 优化模型输出的实体合并、别名识别和跨章节关系稳定性。
- 将选中文本菜单里的“解惑”“回忆”“出处”等操作接入真实 AI 工作流。
- 改进人物关系、组织架构、地点路径和时间线的可视化。
- 完善桌面体验后，再设计移动端阅读体验。
- 将阅读能力树作为后续功能重新设计，避免干扰核心阅读流程。

## 说明

这是一个快速迭代中的产品原型，目标是探索“AI 如何真正降低长篇阅读的认知负担”。当前实现更重视交互方向和阅读体验验证，部分格式解析、AI 行为和可视化能力仍在持续打磨中。
