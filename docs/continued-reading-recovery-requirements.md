# 续读恢复与当前理解需求文档

## 1. 背景

读者在间隔一段时间后重新打开一本书，常见问题不是单纯“不知道读到第几页”，而是无法快速恢复前文情境：前面发生了什么、当前内容承接了哪条线索、哪些信息是理解当前页的前置条件。

本需求基于阅读理解与学习科学中的几个稳定结论：主动回忆有助于长期保持；深层阅读理解依赖情境模型；总结、提问、图形组织器和结构识别有助于阅读理解；间隔复习应根据离开时间动态调整。

因此，本功能不以“展示更多索引”为目标，而以“帮助读者恢复阅读状态并理解当前内容”为目标。

## 2. 目标

在读者重新进入一本书时，系统应在 30 秒内帮助读者完成三件事：

1. 知道自己上次读到哪里。
2. 恢复前文主线和关键上下文。
3. 理解当前页为什么重要，以及理解它需要回忆哪些前置信息。

## 3. 非目标

本需求不追求以下内容：

- 不把人物、地点、时间线作为所有书籍的默认核心。
- 不在打开书时展示长篇 AI 总结。
- 不把关系图作为默认入口。
- 不把书签和笔记本身视为记忆恢复核心，除非它们被转化为可回忆问题或证据锚点。
- 不因为现有功能存在，就强行把“续读恢复”解释成现有功能的组合。

## 4. 用户场景

### 4.1 短时间离开后继续阅读

读者昨天读过一本书，今天重新打开。系统应给出很轻的承接信息，避免打断阅读。

### 4.2 多天后重新打开

读者隔了三天以上重新打开。系统应提供更完整的前文关键点、当前页前置理解和一个主动回忆问题。

### 4.3 长时间未读后恢复

读者隔了一周以上重新打开。系统应优先帮助读者重建主线：上一阶段发生了什么、主要冲突或观点是什么、当前内容接在哪里。

### 4.4 当前内容读不懂

读者正在阅读当前页，但不知道某句话、某个事件或某个概念为什么重要。系统应解释它与前文的关系，并给出可追溯证据。

## 5. 功能需求

### 5.1 续读恢复卡

当读者打开一本有阅读历史的书时，系统应展示一个轻量的“续读恢复卡”。

恢复卡至少包含：

1. 上次读到的位置。
2. 前文关键点 3 条。
3. 当前页前置理解 2 条。
4. 一个主动回忆问题。
5. 可展开的原文证据。

恢复卡应短、清楚、可跳过，不应阻塞正文阅读。

### 5.2 主动回忆

系统应在续读时提供一个轻量问题，用于唤醒读者记忆。

问题示例：

- “你还记得上一阶段的主要矛盾是什么吗？”
- “这个人物上次出现时处在什么状态？”
- “当前事件为什么会发生？”
- “上一节最后留下了什么问题？”

读者可以选择：

- 自己想一下
- 查看提示
- 直接看答案

主动回忆问题应根据书籍类型生成，不应固定套用人物或事件。

### 5.3 当前页前置理解

系统应为当前页生成“理解前置条件”。

每条前置理解应回答：

- 当前内容承接了前文什么信息？
- 如果不记得这个信息，会影响理解哪里？
- 它为什么值得在当前页被提醒？

示例：

- 历史/传记：当前事件承接了前面某次政治变化或军事行动。
- 小说：当前对话承接了之前的人物关系或冲突。
- 科普：当前概念依赖前面某个机制或定义。
- 技术书：当前代码依赖前面某个接口、流程或约束。

### 5.4 证据型回顾

所有 AI 生成的关键点、前置理解、主动回忆答案和关系说明，都必须支持原文溯源。

每条回顾信息至少包含：

- 结论
- 重要性说明
- 章节/页码/段落位置
- 原文片段

如果无法找到可靠证据，不应生成确定性结论。

### 5.5 类型自适应记忆锚点

系统应根据书籍类型选择最有助于恢复理解的信息，而不是固定抽取人物、地点、时间线。

建议类型锚点：

| 书籍类型 | 优先记忆锚点 |
| --- | --- |
| 小说 / 文学 | 人物、关系、冲突、情节线、场景变化、主题意象 |
| 传记 / 历史 | 人物、组织、事件、地点、时间线、因果链 |
| 科普 | 概念、机制、实验、例子、误区、因果关系 |
| 商业 / 管理 | 模型、案例、决策、指标、行动建议、反例 |
| 哲学 / 思想 | 命题、概念区分、论证链、反驳对象、思想流派 |
| 技术 / 编程 | 概念、流程、依赖、代码片段、常见问题、实现约束 |
| 教材 / 学习 | 定义、例题、易错点、练习、前置知识、方法 |

系统应允许模型根据具体书籍再调整锚点。

### 5.6 图形组织器按需出现

系统不应默认展示关系图、时间线或概念图，而应根据内容结构和书籍类型决定是否出现。

触发建议：

- 人物/组织关系复杂时，展示关系图。
- 事件顺序重要时，展示时间线。
- 概念依赖复杂时，展示概念图。
- 观点推理复杂时，展示论证链。

图形组织器必须可追溯到原文证据。

### 5.7 离开时间自适应

系统应根据读者离开时间调整恢复强度。

| 离开时间 | 恢复策略 |
| --- | --- |
| 1 天内 | 简短承接，少打扰 |
| 1 到 3 天 | 前文关键点 + 当前页前置理解 |
| 3 到 7 天 | 主动回忆 + 关键点 + 前置理解 |
| 7 天以上 | 主线摘要 + 主动回忆 + 关键证据 + 必要图形组织器 |

用户应能关闭或跳过恢复卡。

## 6. 数据需求

系统需要记录以下数据：

- 每本书的最近阅读位置。
- 最近阅读时间。
- 已读范围边界。
- 上次恢复卡生成时间。
- 已生成的恢复摘要与证据。
- 用户是否跳过、展开证据、点击答案、继续阅读。
- 书籍类型与当前可用记忆锚点。

数据应按书籍维度隔离。

## 7. AI 分析要求

AI 分析必须遵守以下原则：

- 只分析已读范围，不剧透未读内容。
- 输出必须结构化。
- 每条关键结论必须尽量绑定证据。
- 根据书籍类型选择锚点。
- 输出应短，优先帮助恢复理解，不追求百科式完整。
- 不确定时应降低语气，或提示证据不足。

## 8. 交互要求

续读恢复卡应满足：

- 可跳过。
- 不遮挡正文主阅读区域太久。
- 允许用户一键继续阅读。
- 支持展开证据，但默认不展开。
- 支持“查看提示/查看答案”的主动回忆流程。
- 视觉风格应延续当前阅读器的简洁、克制、图标优先风格。

## 9. 验收标准

### 9.1 基础验收

- 有阅读历史的书重新打开时，能看到续读恢复卡。
- 恢复卡显示正确的上次阅读位置。
- 恢复卡不包含未读范围内容。
- 恢复卡包含至少 3 条前文关键点。
- 恢复卡包含至少 2 条当前页前置理解。
- 恢复卡包含 1 个主动回忆问题。
- 每条关键点可展开原文证据。

### 9.2 类型适配验收

- 历史/传记类书籍可以出现人物、地点、时间线。
- 科普类书籍不应强行展示人物和地点，而应优先展示概念、机制、实验或误区。
- 技术类书籍应优先展示概念、流程、依赖和代码相关锚点。
- 系统在不同类型书籍中的恢复卡结构可以不同，但目标一致。

### 9.3 质量验收

- 恢复内容应短，不超过用户 30 秒可读完的长度。
- 关键点应与当前阅读理解相关，而不是随机出现的实体。
- 不应出现无证据、无意义或明显错误的实体。
- 用户跳过恢复卡后，应能立即进入阅读。

## 10. MVP 建议

第一版只做“续读恢复卡”，不急着做复杂图形组织器。

MVP 范围：

1. 记录最近阅读时间。
2. 打开书时判断离开时间。
3. 基于已读范围生成恢复卡。
4. 恢复卡包含位置、关键点、前置理解、主动回忆问题和证据。
5. 支持跳过、继续阅读、展开证据。

暂不做：

- 复杂主动答题评分。
- 自动间隔复习提醒。
- 多种图形组织器。
- 笔记自动转复习题。

## 11. 后续迭代

后续可以逐步加入：

- 根据用户跳过率调整恢复卡长度。
- 把用户笔记转成主动回忆问题。
- 基于阅读间隔自动调整恢复策略。
- 根据不同书籍类型提供不同图形组织器。
- 建立恢复效果评估系统，例如用户是否继续阅读、是否展开证据、是否返回前文。
